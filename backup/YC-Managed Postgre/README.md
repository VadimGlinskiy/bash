# Бэкап и восстановление YC Managed PostgreSQL

В папке два скрипта:

- `pg_dump.sh` - снять дамп базы в файл `<db_name>.dump`
- `pg_restore.sh` - восстановить базу из локального дампа или из S3

Оба скрипта работают с Managed PostgreSQL в Yandex Cloud (порт `6432`).

## Что нужно перед запуском

- Linux + `bash`
- доступ к вашей БД
- `wget`
- права на `apt install` (если утилиты не стоят)

Скрипты сами проверяют зависимости и при необходимости ставят:

- `pg_dump.sh` -> `postgresql-client`
- `pg_restore.sh` -> `postgresql-client` и `awscli`

Сделайте файлы исполняемыми:

```bash
chmod +x pg_dump.sh pg_restore.sh
```

## Важный момент по `pg_dump.sh`

В `pg_dump.sh` есть плейсхолдеры:

- `HOSTNAME_PLACEHOLDER`
- `URL_PLACEHOLDER`

Их нужно заменить на реальные значения (имя cert-файла и URL сертификата). Иначе TLS-подключение не поднимется.

## `pg_dump.sh` (снять дамп)

Запуск:

```bash
./pg_dump.sh
```

Что спросит скрипт:

- хост БД (сохранится в `~/.db_host`)
- имя базы
- пароли пользователей:
  - `<db_name>-app`
  - `<db_name>-admin`
  - `<db_name>-user`

Как работает:

- по очереди пытается подключиться каждым из пользователей
- при первом успешном подключении делает дамп
- пишет файл в текущую директорию: `<db_name>.dump`

Если ни одним пользователем подключиться не получилось, скрипт завершится с ошибкой.

## `pg_restore.sh` (восстановить дамп)

Запуск:

```bash
./pg_restore.sh
```

Скрипт:

- берет хост БД из `./db_host` (или спросит и сохранит)
- предлагает источник дампа:
  - локальный файл
  - скачать из S3
- предлагает имя БД по имени dump-файла (можно принять Enter или ввести свое)
- пробует восстановить по очереди от пользователей:
  - `<db_name>-app`
  - `<db_name>-admin`
  - `<db_name>-user`

Если выбрано S3:

- бакет берется из `./s3_bucket` (или спрашивается и сохраняется)
- файл качается через `aws s3 cp` с endpoint `https://storage.yandexcloud.net`
- регион: `ru-central1`
- после восстановления скачанный dump удаляется

Для S3 заранее должен быть настроен `~/.aws/credentials`.

## Где скрипты хранят настройки

- `pg_dump.sh` -> `~/.db_host`
- `pg_restore.sh` -> `./db_host`
- `pg_restore.sh` (S3 bucket) -> `./s3_bucket`

Обрати внимание: у dump и restore разные файлы для хоста БД.
