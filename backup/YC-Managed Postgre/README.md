# pg_dump.sh

Скрипт для создания дампа базы данных в Yandex Cloud Managed PostgreSQL.

## Что делает скрипт

- Проверяет наличие `pg_dump`.
- При отсутствии пытается установить `postgresql-client` через `apt`.
- Берет хост БД из файла `~/.db_host` или запрашивает его у пользователя.
- Загружает корневой сертификат YC в `~/.postgresql/HOSTNAME_PLACEHOLDER` (если его нет).
- Запрашивает имя базы и пароли для пользователей:
  - `<db_name>-app`
  - `<db_name>-admin`
  - `<db_name>-user`
- Последовательно пытается выполнить `pg_dump` на порту `6432` с `sslmode=verify-full`.
- Сохраняет дамп в файл `<db_name>.dump` в текущем каталоге.

## Требования

- Linux с `bash`.
- Доступ к кластеру Managed PostgreSQL.
- Права на установку пакетов через `apt` (если `pg_dump` не установлен).
- `wget` для загрузки сертификата.

## Запуск

```bash
chmod +x pg_dump.sh
./pg_dump.sh
```

## Вводимые данные

- Хост базы данных (один раз, затем хранится в `~/.db_host`).
- Имя базы данных.
- Пароли для пользователей `<db_name>-app`, `<db_name>-admin`, `<db_name>-user`.

## Результат

- При успешном подключении создается файл дампа `<db_name>.dump`.
- Если все попытки авторизации неуспешны, скрипт завершается с ошибкой.
